# UI Guidelines

## General rules

1. One spacing system.
2. One card style.
3. One button style per semantic role.
4. One status-badge system.
5. Avoid excessive gradients and shadows.

## Mobile-first rules

- Prioritize team names over labels.
- Avoid long status chips inside narrow match cards.
- Use icons and subtle opacity instead of verbose text when space is limited.

## Stats cards

Stats cards may be clickable when they reveal top-3 or detail panels. Non-clickable and clickable stats should still have equally bright icons and labels.

## Fantasy cards

Fantasy cards should be chronological and should clearly distinguish:

- confirmed teams
- provisional teams
- TBD matchups
- locked predictions
- editable predictions

## Actual Bracket

Actual Bracket should be the visual reference for confirmed/provisional team display. Other features should reuse the same language where practical.
